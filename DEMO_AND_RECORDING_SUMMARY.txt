MANUAL RECORDING (If Script Doesn't Work)
═══════════════════════════════════════════════════════════════════════════════

Option A - Simple FFmpeg Command:
  ffmpeg -f x11grab -s 1920x1080 -r 30 -i :0 -c:v libx264 -crf 23 \
    -preset veryfast demo.mp4
  [Then press 'q' to stop]

Option B - With Audio:
  ffmpeg -f x11grab -s 1920x1080 -r 30 -i :0 -f pulse -ac 2 -i default \
    -c:v libx264 -c:a aac demo_audio.mp4

Option C - SimpleScreenRecorder GUI:
  simplescreenrecorder
  [Configure and click Start Recording]

Option D - GNOME Recorder:
  Ctrl+Alt+Shift+R (Start)
  Ctrl+Alt+Shift+R (Stop)
  [Video saved to ~/Videos/]

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Issue: Recording script doesn't start
  Solution: Check if FFmpeg is installed
    sudo apt-get install -y ffmpeg

Issue: Application not responding
  Solution: Check servers are running
    curl http://localhost:8000/health
    curl http://localhost:8501

Issue: Recording uses too much CPU
  Solution: Use faster preset
    ffmpeg -f x11grab -s 1920x1080 -r 30 -i :0 \
      -c:v libx264 -preset ultrafast demo.mp4

Issue: File too large
  Solution: Increase compression
    ffmpeg -i demo.mp4 -c:v libx264 -crf 30 demo_small.mp4
